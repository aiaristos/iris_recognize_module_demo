    API 變更日誌

    [v.beta.2024.09.26] - 2024-09-26

    變更

    • 在 開始虹膜建檔 新增了 need_export 可以用於設定是否額外輸出該虹膜的紋路圖，預設為true
    • 在 .env.secrect 中新增了 USE_EMA_UPDATE_METHOD 該參數可以用於控制是否使用EMA的方式去"動態"更新檢測是否眨眼或晃動的模糊度閾值，可更好的去適應各種光源與環境

    [v.beta.2024.06.20] - 2024-06-20

    變更

    • 在 開始虹膜比對 API中新增了export_pdf_path，可指定當比對失敗時，要輸出的PDF資料夾路徑，預設./exportCompareFailedPDF，即產生在此模組資料夾中
    • 在 開始虹膜比對 API比對失敗後，會產生一份檔名為{loftId}_{feetNumber}_{yyyyMMddHHmmss}.pdf的pdf 繁體範例、簡體範例
    • 在 env 新增了 PDF_EXPORT_LANG 可以用於設定是否指定pdf輸出的語言與編碼，預設為系統自動偵測
    • 在 開始虹膜建檔 每次建檔後，會額外輸出該虹膜的紋路圖供保存，預設路徑如下{folder_path}/export_{season_id}/{loft_id}，而當次檔名則為{loft_id}_{feet_number}_{當次建檔產生之檔名}.jpg

    [v.beta.2024.05.24] - 2024-05-24

    變更

    • 新增了 .env.secrect 的相關描述文檔
    • 新增 setUseNewMethod API，在 建檔 或 比對 的當下，發現無法成功進行可透過此API去調整是否切換至舊版分析方法
    • 新增了建檔比對模糊度閾值， 可參考 .env.secrect

    [v.beta.2024.05.20] - 2024-05-20

    變更

    • 新增了建檔、比對錯誤訊息的主動回傳 當建檔和比對有未歸檔的錯誤發生時，會傳送此訊息給使用者告知

    [v.beta.2024.05.10] - 2024-05-10

    變更

    • 新增了env 的 SHOW_EYE_DETECT_UNAVAILABLE_BOX 用於控制是否顯示紅色框
    • 新增了env 的 SHOW_PROGRESS_BAR 用於控制是否顯示建檔的進度條
    • 新增了重新載入env設定API，方便在調整完env檔案後，不用關閉程式可直接重新載入設定內容

    [v.beta.2024.04.12] - 2024-04-12

    變更

    • 新增了左右翻轉攝影機API flipCamera，可傳入camera_src = 'left' or camera_src = 'right' 來決定要翻轉的鏡頭對象。
    • 在 env 新增了左右鏡頭各自的flip 設定，方便在程式下次開啟時自動翻轉鏡頭。程式重啟後生效
    • 新增了建檔提示畫面, 建檔進度條
    • 新增了關閉建檔完成提示API closeHint，用於關閉建檔提示。
    • 新增了隱藏建檔進度條API closeProgressbar, 用於以外部關閉建檔的進度條。
    • 將建檔擷取的影像以及分析檔，在該檔名前面加上L-, R- 來區分是由左側還是右側的鏡頭所產生的檔案。

    [v.beta.2024.04.02] - 2024-04-02

    變更

    • 新增了主動回傳載入完成的訊息功能到您的Name Pipe Server 中，請參見載入任務回傳結果
    • 在所有API回傳格式中都新增 Action 欄位，方便去辨別此回傳是由哪個Action觸發的
    • 調整了 開始虹膜建檔API startBuildIris
      • 現在會判斷left_build_num/right_build_num 是否為0來決定要關閉的鏡頭窗口，可更方便指定左右各自的建檔數量。
        • 移除了build_num 欄位
        • 新增了left_build_num 欄位
        • 新增了right_build_num 欄位
      • 現在會自動判斷左右兩側的建檔數量加總是否為0，如果是，則不予建檔並回傳錯誤訊息
    • 將原 set_focus_camera API 調整為 set_compare_focus_camera API
      • 移動到比對模式底下，讓比對時可以指定只使用單一邊的鏡頭去做比對
    • 新增了禁止使用此攝影機的畫面
    • 現在可在開啟攝影機API額外傳入single_camera_mode，來決定是否要開啟單鏡頭模式，以應對緊急情況
    • 移除了env中的theme設定，只保留會使用的單一主題

    [v.beta.2024.03.20] - 2024-03-20

    變更

    • 新增了比對虹膜的相關API

    • 在建檔與比對的回傳結果中新增了 Action 參數，方便判斷程式主動拋出的是哪個動作

    • 配合雙鏡頭做出以下API參數相關調整

      • 預設建檔張數從 3 -> 6 張
      • 移除了使用左眼的參數
      • 新增 set_focus_camera api 讓如果臨時有單邊使用的需求也能正常使用

    • 新增了畫面狀態說明 ，裡面說明了各個畫面代表的狀態

    • 新增了API詳細應用說明 ，裡面針對API的一些應用做了相對應的補充

    [v.beta.2024.01.12] - 2024-01-12

    變更

    • 配合嵌入式做出程式內調整
      • 為了讓.Net 可以正確獲取python 程式的MainWindowHandle，現已預設關閉攝影機的自動開啟。 等到.Net 成功嵌入後再發送開啟攝影機訊號開啟即可
      • 配合.Net winForm 可能無法正確render 透明區域的GUI，導致無法如期顯示，已配合修改GUI寫法做出調整。
    • 新增以下API
      • 關閉程式
      • 開啟攝影機
      • 關閉攝影機
    • 新增以下環境變數
      • 在 theme 中 新增了 custom_title_bar 可供選取。此樣式移除所有windows應用程式外部控制框 僅保留鏡頭畫面。
      • custom_title_bar 以在此版本更改為預設樣式。
