    API 相關應用詳細說明

    在您對python 的NamePipe Server 端發送指令後，當內部程式處理完會立即傳送指令執行結果給客戶端，以下針對各情況做更詳細的說明。 以下會以 .Net 作為連接的客戶端做示範

    1. 關於應用

    建立連接
        // 與python 建立起來的NamedPiped Server 做連線
    // "." 指定的伺服器名稱，此處固定用"."即可
    // "PythonPipe" 在本軟體中所使用的管道名稱，可在".env"檔案中做修改
    // PipeDirection.InOut 指定管道為雙向，代表可讀寫
    NamedPipeClientStream pipeClient = new NamedPipeClientStream(".", "PythonPipe", PipeDirection.InOut);

    // 建立向指定管道寫入資料的物件
    // pipeClient .net建立的client
    // Encoding.UTF8 編碼
    // -1 buffer 大小
    // leaveOpen 在做完通訊後是否關閉與管道的連接
    StreamWriter sw = new StreamWriter(pipeClient, Encoding.UTF8, -1, leaveOpen: true);

    // 設定在資料寫入後將其緩衝區清除
    // 這樣寫可以讓python api 可以馬上接受到您傳送的指令
    // 如果是false 則需要等待您的sw close後才會執行要求的指令
    sw.AutoFlush = true;

    // 建立從指定管道讀取資料的物件
    // pipeClient .net建立的client
    // Encoding.UTF8 編碼
    // true 是否在檔案開頭尋找位元順序標記
    // -1 buffer 大小
    // leaveOpen 在做完通訊後是否關閉與管道的連接
    StreamReader sr = new StreamReader(pipeClient, Encoding.UTF8, true, -1, leaveOpen: true);

    2. 關於讀寫

    以下介紹如何定義要傳送的資料格式，以及傳送方法與是否採用阻塞並接收

    資料格式
        // 在python 預設接收json格式的資料
    // 此處先定義了一個Operation的class作為示範
    // Action 定義要使用的對應api接口
    // Data 在該接口中所要求的相關資料
    Operation operation = new Operation
    {
        Action = "startBuildIris",
        Data = new
        {
            season_id = seasonIdTextBox.Text,
            loft_id = loftIdTextBox.Text,
            feet_number = feetNumberTextBox.Text,
            folder_path = @"C:\iris_data"
        }
    };

    // 轉換成json
    string message = JsonSerializer.Serialize(operation);

    傳送與接收
        // 在python 端接收到指令後
    // 會立即回傳該指令是否運行成功的json格式之結果
    // 這裡可以選擇是否要阻塞程式等待結果回傳
    // 或者選擇不等待，但並不推薦此種方式
    // !如果不接收的話則需要考慮該如何處理資料錯位的問題

    // 送出資料
    sw.WriteLine(message);

    // 阻塞並等待結果回傳
    // 使用StreamReader讀取資料時
    // 必須等到有實際讀取到資料才會繼續往下執行
    // 如果有運用上的考量，也可不寫此行，但要注意資料錯位
    // 比如會取得前幾次的回傳結果等情況
    // 推薦使用異步或是執行緒的方式來處理
    string result = sr.ReadLine();

    // 由於回傳結果預設也是json格式
    // 將結果轉換回obj方便做後續判讀
    RecivedResult recivedResult = JsonSerializer.Deserialize<RecivedResult>(result);

    // 處理資料錯位範例(不推薦)
    while (recivedResult.Action != "startBuildIris")
    {
        // 不斷取得直到取得相同的Action回應為止
        result = sr.ReadLine();
        recivedResult = JsonSerializer.Deserialize<RecivedResult>(result);
    }

    3. 關於模式

    考慮到某些API在運行時，彼此可能會有行為上的衝突 比如: 在建檔時，比對的相關功能都不該允以使用 又比如: 在比對時，不該可關閉鏡頭或是類似的GUI操控

    所以，在系統中將API分成了三種模式 分別為

    normal: 正常模式，可做一些基本的GUI行為操縱
    build_iris: 建檔模式，包含了建檔的相關行為
    compare_iris: 比對模式，包含了比對的相關行為

    以下表格列出在各模式下可使用的相關api

    模式 可用API
    正常模式 setPosition
    setFrameSize
    setActivated
    setMinimized
    closeProgram
    openCamera
    closeCamera
    flipCamera
    closeHint
    closeProgressbar
    startBuildIris: 呼叫後會進入建檔模式
    startCompareIris: 呼叫後會進入比對模式
    建檔模式 setActivated
    setMinimized
    closeProgram
    stopBuildIris: 呼叫後會脫離建檔模式

    建檔完畢後會自動脫離返回正常模式
    比對模式 setActivated
    setMinimized
    closeProgram
    setCompareFocusCamera
    stopCompareIris: 呼叫後會脫離比對模式

    比對完畢後會自動脫離返回正常模式

    如果系統判斷當前模式不支援此接口，則會回傳以下訊息

    Response Sample
        {
        "Status": "error",
        "StatusCode": 1,
        "Message": "Action '{您傳入的Action}' is not supported in mode '{當前的模式}'"
    }

    4. 關於資料驗證

    某些接口在介接上有資料驗證的需求，以下列出這些接口以及會檢查的內容

    接口 驗證內容說明
    setPosition x: x座標，限制為int型態的數值，系統內會嘗試轉換型態，預設數值0
    y: y座標，限制為int型態的數值，系統內會嘗試轉換型態，預設數值0
    setFrameSize width: 程式寬度，限制為int型態的數值，系統內會嘗試轉換型態，預設數值100
    height: 程式高度，限制為int型態的數值，系統內會嘗試轉換型態，預設數值100
    openCamera single_camera_mode: 選擇採用的單鏡頭模式，可針對緊急情況下調整到單鏡頭模式
    限制只能傳入left, right，和disable 這三個string字串，預設數值disable

    disable: 一般雙鏡頭模式
    left, right: 只開啟左或右側的鏡頭
    flipCamera camera_src: 要指定的camera來源，要求系統去翻轉的鏡頭目標
    限制只能傳入left, right 限制為string型態的字串，必填
    closeProgressbar camera_src: 要指定的camera來源，要求系統去關閉哪個鏡頭底下的進度條
    限制只能傳入left, right, all 這三個string字串，預設數值all
    startBuildIris season_id: 賽季編號，限制為string型態的字串，必填
    loft_id: 舍號，限制為string型態的字串，必填
    feet_number: 腳環編號，限制為string型態的字串，必填
    folder_path: 建檔資料夾，限制為string型態的字串，必填
    left_build_num: 左側建檔張數，限制為int型態的數值，系統內會嘗試轉換型態，預設數值3
    right_build_num: 右側建檔張數，限制為int型態的數值，系統內會嘗試轉換型態，預設數值3

    注意!! 此處還會額外檢查 左側和右側建檔張數加總是否大於0
    startCompareIris season_id: 賽季編號，限制為string型態的字串，必填
    loft_id: 舍號，限制為string型態的字串，必填
    feet_number: 腳環編號，限制為string型態的字串，必填
    folder_path: 建檔資料夾，限制為string型態的字串，必填
    threshold: 比對門檻值，限制為float型態的數值，系統內會嘗試轉換型態，傳入區間限制0.30 ~ 0.50，預設數值0.35，數值越低越嚴格
    compare_times: 要比對的次數，限制為int型態的數值，系統內會嘗試轉換型態，傳入區間限制 >= 1 ，預設數值10
    need_save: 是否儲存比對通過的結果，限制為boolean
    setCompareFocusCamera camera_src: 要指定的camera來源，要求系統辨識時只能使用畫面上左側或是右側的鏡頭
    限制只能傳入left, right 這兩個string字串，預設數值right
